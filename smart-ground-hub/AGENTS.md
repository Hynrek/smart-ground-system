# AGENTS.md — smart-ground-backend

> For AI agents and automated tooling. Describes architecture, conventions, and key patterns for safe codebase navigation.

---

## Project Overview

**Smart Ground** (SmartRange) is an IoT management system for automated clay pigeon shooting ranges:
- REST API for ranges, devices, device types, and SmartBoxes
- MQTT integration for real-time communication with Raspberry Pi Pico 2W devices ("SmartBoxes")
- PostgreSQL persistence via Liquibase migrations

## Package Structure

```
ch/jp/shooting/
├── api/          # REST controllers (thin — no business logic) + GlobalExceptionHandler
├── config/       # MQTT wiring, push service, CORS, Jackson
├── model/        # JPA entities
├── repository/   # Spring Data JPA repositories
├── exception/    # Custom exceptions (all registered in GlobalExceptionHandler)
├── mapper/       # EntityMappers: JPA entity → response DTO
├── service/      # Currently empty (package-info.java only) — add new logic here
└── SmartGroundApplication.java   # @EnableScheduling entry point
```

### ⚠️ Two distinct Java packages

| Package | Origin | Contents |
|---|---|---|
| `ch.jp.shooting.*` | Handwritten | Controllers, entities, repos, etc. |
| `ch.jp.smartground.*` | Generated by openapi-generator | Spring interfaces (`api`) + request/response DTOs (`model`) |

Controllers in `ch.jp.shooting.api` **implement** the generated interfaces from `ch.jp.smartground.api`. Never confuse the two.

## API — Interface-First

- Spec: `src/main/resources/static/openapi.yaml`
- Generator writes to `target/generated-sources/openapi/…` — added as source root via `build-helper-maven-plugin`
- **Always update `openapi.yaml` first**, then run `mvn generate-sources`, then implement
- `openapi-generator-ignore` controls which generated files are preserved across runs

> **Exception:** `DeviceController` has manually annotated `@GetMapping`/`@PostMapping` endpoints (e.g. `/{id}/command`) not yet in the spec. These should become spec-first.

## MQTT Architecture

### Topics

**Inbound (SmartBox → Backend):**

```
smartboxes/{mac}/discovery   → boot announcement; triggers upsert + immediate config push
smartboxes/{mac}/status      → health/state update
smartboxes/{mac}/config/ack  → acknowledges config push
```

**Outbound (Backend → SmartBox):**

```
smartboxes/{mac}/config      → device-to-GPIO config payload
smartboxes/{mac}/command     → fire/trigger command
```

> `devices/ack` was planned but has no handler — `SmartBoxMqttRouter` only routes `discovery`, `status`, and `config/ack`.

### Payloads

**Discovery (inbound):**
```json
{ "mac": "AA:BB:CC:DD:EE:FF", "ip": "192.168.1.42", "firmwareVersion": "1.0.3" }
```

**Config push (outbound):**
```json
{ "devices": [{ "id": "<uuid>", "alias": "Werfer 1", "gpioPin": 14, "signalType": "LED", "defaultSignalDurationS": 1 }] }
```

### Key MQTT Classes (`config/`)

| Class | Role |
|---|---|
| `MqttConfig` | Spring Integration channels + Paho adapters |
| `MqttCommandPublisher` | Publishes to any topic |
| `SmartBoxMqttRouter` | Routes inbound messages by topic suffix |
| `SmartBoxDiscoveryHandler` | Upserts SmartBox, triggers config push |
| `SmartBoxStatusHandler` | Updates SmartBox state |
| `SmartBoxConfigAckHandler` | Handles config acknowledgement |
| `SmartBoxConfigPushService` | Builds + publishes device-to-GPIO config |
| `CorsConfig` | Reads `cors.allowed-origins` |
| `JacksonConfig` | Custom serialization — check before adding `@JsonProperty` |

## Domain Model

### `SmartBox` (table: `smart_boxes`)

| Field | Type | Notes |
|---|---|---|
| `macAddress` | `String` | Unique hardware identifier |
| `alias` | `@Nullable String` | Human-readable label |
| `status` | `SmartBoxStates` | `@Enumerated(EnumType.STRING)` — values: `OFFLINE`, `ONLINE`, `BLOCKED` |
| `lastSeen` | `@Nullable Instant` | Updated on discovery/status |
| `firmwareVersion` | `@Nullable String` | From discovery payload |
| `mqttUsername` | `@Nullable String` | Optional MQTT auth |
| `configSynced` | `boolean` | `false` while config push is pending |

### `Device` (table: `devices`)

| Field | DB Column | Notes |
|---|---|---|
| `smartBox` | `smartbox_id` | FK to `smart_boxes` |
| `deviceType` | `device_type_id` | FK to `device_types` |
| `range` | `range_id` | Nullable FK to `ranges` |
| `pinConfig` | `pin_config` | JSONB |
| `fireDelayMs` | `fire_delay_ms` | Pulse duration in ms |
| `blocked` / `healthy` | — | Booleans |

### `DeviceType` (table: `device_types`)

| Field | Notes |
|---|---|
| `gpioPin` | ⚠️ Physical GPIO pin — included in config push payload |
| `signalType` | `DeviceSignalType` enum: `LED`, `INPUT` |
| `boxType` | Firmware handler name |
| `defaultSignalDurationS` | Included in config push payload |

Seed data (Liquibase): `Werfer` (TRAP), `LED`, `Knopf` (BUTTON), `Sensor`

### `Range` (table: `ranges`)

`name` (unique), `description`, `locked`, `createdAt` (`LocalDateTime`, serialized as UTC `OffsetDateTime`), lazy `devices` list.

### Mappers

All API responses go through `mapper/EntityMappers.java`. Never expose raw JPA entities.

## Database

- Schema managed **exclusively by Liquibase** — never set `ddl-auto=create/update`
- Changelog: `src/main/resources/db/changelog/changelog-master.xml`
- Always add schema changes as new versioned files (e.g. `v1.1__add_column.xml`) — never edit existing ones
- FK naming convention: `{entity}_{field}` in snake_case (e.g. `smartbox_id`, `device_type_id`, `range_id`)

## Coding Conventions

- **Null safety**: `@NullMarked` on every package (via `package-info.java`). Use `@Nullable` for optional fields.
- **No logic in controllers**: delegate to `service/` or `config/` handlers.
- **API-first**: update `openapi.yaml` → `mvn generate-sources` → implement.
- **Exceptions**: extend `RuntimeException`, add to `exception/`, register in `GlobalExceptionHandler` with a `/errors/{slug}` type URI.
- **Jackson**: check `JacksonConfig` before adding `@JsonProperty`.
- **German terminology**: identifiers in English; comments/docs may use German domain terms (Schütze, Platz, Wurfmaschine, etc.).

## Common Agent Tasks

**New REST endpoint:**
1. Edit `openapi.yaml` → `mvn generate-sources`
2. Implement in `api/` controller
3. Add exception(s) in `exception/` + `GlobalExceptionHandler`
4. Add mapper logic in `EntityMappers`

**New MQTT handler:**
1. Create handler class in `config/`
2. Register in `SmartBoxMqttRouter` (match on `topic.endsWith("/your-topic")`)
3. Document the topic above

**New DB table:**
1. New Liquibase file in `db/changelog/`, referenced from `changelog-master.xml`
2. JPA entity in `model/`, repository in `repository/`
3. Use explicit `@Column(name = "…")` — do not rely on Hibernate naming defaults

**Send device command via MQTT:**
```java
String topic = "smartboxes/%s/command".formatted(device.getSmartBox().getMacAddress());
mqttCommandPublisher.publishToTopic(topic, payload);
```
